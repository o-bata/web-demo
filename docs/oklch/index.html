OKLCH カラーモデルの説明

OKLCH は、色を Oklch(L C H) の形式で指定するカラーモデルです。
より人間の視覚に近い形で色の距離を表現でき、カラーアクセシビリティに優れています。

  • L: 明度（0〜1）
  • C: 彩度（Chroma）
  • H: 色相（Hue、度数 0〜360）

CSSでは、以下のように指定できます:

color: oklch(0.75 0.2 40);

他の色空間との比較

代表的な色空間として以下のものがあります：

  • RGB: 赤・緑・青の強さで色を表す。モニターでの表示に最適だが、人の視覚的な差異とは一致しにくい。
  • HSL: 色相・彩度・明度。直感的だが色のバランスに偏りが出やすい。
  • OKLCH: 視覚均等性に基づく設計で、彩度や明度を変化させた時に自然な色の変化が得られる。

色相変化の統一性

OKLCHは、同じ明度と彩度で色相だけを変化させた場合でも、色の鮮やかさや明るさが統一されるという特徴があります。以下の例で比較してみましょう。

OKLCH 色相変化 (L=0.6, C=0.2)

oklch(0.6 0.2 0)    /* 赤 */
oklch(0.6 0.2 60)   /* 黄 */
oklch(0.6 0.2 120)  /* 緑 */
oklch(0.6 0.2 180)  /* シアン */
oklch(0.6 0.2 240)  /* 青 */
oklch(0.6 0.2 300)  /* マゼンタ */

HSL 色相変化 (L=60%, S=50%)

hsl(0, 50%, 60%)    /* 赤 */
hsl(60, 50%, 60%)   /* 黄 */
hsl(120, 50%, 60%)  /* 緑 */
hsl(180, 50%, 60%)  /* シアン */
hsl(240, 50%, 60%)  /* 青 */
hsl(300, 50%, 60%)  /* マゼンタ */

まとめ

OKLCHは、デザインやUI設計において色の統一性・調整のしやすさを高めるための新しいカラーモデルです。アクセシビリティや色の再現性を重視する場面で特に有効です。
