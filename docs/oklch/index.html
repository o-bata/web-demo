OKLCH カラーモデルの説明

OKLCH は、色を Oklch(L C H) の形式で指定するカラーモデルです。
より人間の視覚に近い形で色の距離を表現でき、カラーアクセシビリティに優れています。

  • L: 明度（0〜1）
  • C: 彩度（Chroma）
  • H: 色相（Hue、度数 0〜360）

CSSでは、以下のように指定できます:

color: oklch(0.75 0.2 40);

他の色空間との比較

代表的な色空間として以下のものがあります：

  • RGB: 赤・緑・青の強さで色を表す。モニターでの表示に最適だが、人の視覚的な差異とは一致しにくい。
  • HSL: 色相・彩度・明度。直感的だが色のバランスに偏りが出やすい。
  • OKLCH: 視覚均等性に基づく設計で、彩度や明度を変化させた時に自然な色の変化が得られる。

グラデーションの美しさ

OKLCHはグラデーションを作成する際にも優れた特性を発揮します。以下の例では、CSSのグラデーションを使用して比較しています。

OKLCH グラデーション

background: linear-gradient(to right, oklch(0.4 0.3 120), oklch(0.7 0.3 120));

HSL グラデーション

background: linear-gradient(to right, hsl(120, 80%, 40%), hsl(120, 80%, 70%));

OKLCH マルチカラーグラデーション

background: linear-gradient(to right, oklch(0.5 0.3 120), oklch(0.6 0.3 200), oklch(0.5 0.3 280));

HSL マルチカラーグラデーション

background: linear-gradient(to right, hsl(120, 80%, 50%), hsl(200, 80%, 60%), hsl(280, 80%, 50%));

OKLCHのグラデーションは、明度の変化に応じて自然な色の変化を示します。一方、HSLでは明度を変化させると色の鮮やかさが不均一になり、特に暗い部分で色が濁って見えることがあります。これは、OKLCHが人間の視覚特性に基づいて設計されているためです。

OKLCH (L=0.7, H=40)

HSL (H=40°, L=70%)

HSLでは高彩度になるほど色が不自然に感じられることがありますが、OKLCHは彩度を上げても滑らかで調和の取れた色合いになります。

まとめ

OKLCHは、デザインやUI設計において色の統一性・調整のしやすさを高めるための新しいカラーモデルです。アクセシビリティや色の再現性を重視する場面で特に有効です。
